What is ML Works?

ML Works is an enterprise grade model monitoring platform that enables organizations to monitor its ML models at scale in production.

This community version of ML Works is primarily aimed towards Data Scientists and Data Engineers who would like to play around with the tool and also use this tool to try out some of the features like Drift and Explainability.

The Machine Learning lifecycle has the following stages: Build, Deploy, Monitor and Maintain. ML Works aims at showcasing how “model monitoring” can be streamlined and automated, giving precious time back to data scientists to solve new problems. Learn how ML Works helps you identify model degradation over time and get a better understanding of ML output.


How should you use ML Works?

You can use ML works in the following ways:

  • System Projects: Explore ML Works by browsing through our pre-loaded projects (with accessible data and model objects).

    • You can access these projects by going to the Overview tab ( → Icon on the top ribbon ) which gives you a quick summary of the executions.

    • You can also see the results of our Drift detection module which shows you how the test data distribution is different than the training data along with feature contribution to drift summarized in a dashboard.

    • Check out our Explainability module which calculates the feature importance both at a global as well as local level

  • Upload your own dataset: Once you’ve played around with the system projects, you can now try and upload your own dataset to explore some of these features even further.

    • Upload your own data set and model file by click on + icon on the top ribbon.

    • If you would like to understand if your test data has drifted from your training dataset, try running the drift module on the same.

    • If you would like to explain your model predictions better, run the Explainability module on your dataset. You can deep dive into specific data points to understand the exact features that determined the model output. In this case you would also need to upload your model file.

For any further questions you can refer to the FAQ document or reach out to us using our support form in case few of your questions are unanswered.

There is an in-built guided tour which you can activate by clicking on the “Automated Tour” ( → Icon on the top ribbon).


ML Works Capabilities

There are 4 major modules in ML Works:

  • Drift Detection

    Our model agnostic library detects the change in data distribution between the training and production/tagging datasets (called ‘testing data’ within the platform), this indicates the degradation of models due to covariate shift or data drift. To know more about the concepts in use, please refer to the technical documentation of our drift library

  • Explainable AI

    Using the concepts of Shapley values we have created a custom library to enable you to understand model output. The library also allows you to drill down to a single prediction for local explanations. To know more about the concepts in use, please refer to the technical documentation of our Explainable AI library

  • MLOps Graph

    As ML codes in the pipeline can grow upto 100s of lines, it becomes difficult to manage. ML Works transforms the pipeline code into a visual graph to track the execution more efficiently. We aim at making debugging faster and easier to reduce production downtime. For this trial version, our system projects have been equipped with the MLOps Graph. Refer to the 'Graph' column in the overview page to access the MLOps graph.

  • Persona Dashboard

    We believe that in a business there are people from different functions other than Data Scientists who look at the output of a Machine Learning model but through a very different lens. We have broadly broken these users down into 3 personas - Data Scientist, Data Engineer and Business User. Our dashboard generates metrics relevant to each one of these users to clear the clutter for them. Click on on the top ribbon or the play button from the overview page to access the persona dashboard.
